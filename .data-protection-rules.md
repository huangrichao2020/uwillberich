# 数据安全与隐私保护规则

_2026-03-22 21:21 创建_

---

## 🔒 核心规则

**所有个人数据和敏感数据禁止推送到 GitHub**

---

## 📁 受保护的文件分类

### 1. 人格与灵魂数据 (核心身份)
- `SOUL.md` - 核心身份与交易哲学
- `IDENTITY.md` - 交易人格定义
- `memory/` - 所有记忆数据文件

### 2. 用户个人信息
- `USER.md` - 用户姓名、联系方式、偏好等
- `TOOLS.md` - 本地工具配置、API Key、密码等

### 3. 敏感配置
- `.env` - 环境变量 (API Key、密码等)
- `*.env` - 所有环境配置文件
- `runtime.env` - 运行时配置
- `.credentials` - 凭证文件

### 4. 本地规则与指南
- `.github-push-rules.md` - 推送规则
- `.gitignore` - Git 忽略规则本身
- `GITHUB_PUSH_GUIDE.md` - 推送指南 (含认证信息)

### 5. 财务与交易数据
- `analysis/` - 个人分析报告
- `trading_records/` - 交易记录
- `portfolio/` - 持仓数据

### 6. 系统文件
- `.DS_Store` - macOS 系统文件
- `*.swp` - 编辑器临时文件
- `__pycache__/` - Python 缓存

### 推送前检查清单
```bash
# 1. 检查暂存文件
git status

# 2. 检查与远程差异
git diff origin/main

# 3. 确认无敏感文件
# - 检查 USER.md、SOUL.md、IDENTITY.md
# - 检查 memory/ 目录
# - 检查 *.env 文件

# 4. 确认后才推送
git push origin main
```

## ⚠️ 违规处理

### 发现敏感文件被暂存
```bash
# 立即取消暂存
git restore --staged <file>

# 恢复工作区文件
git restore <file>

# 如果已提交但未推送
git reset HEAD~1
```

### 如果已误推送
```bash
# 1. 本地回退
git reset --hard HEAD~1

# 2. 强制推送 (谨慎！)
git push origin main --force

# 3. 如果已泄露，立即：
# - 更改相关密码/API Key
# - 在 GitHub 上删除敏感文件
# - 查看 GitHub audit log
```

## 🛡️ 预防措施

### 1. 使用本地 .gitignore
始终确保 `.gitignore` 包含所有敏感文件模式

### 2. 推送前审查
每次推送前执行：
```bash
git status
git diff --cached --name-only
```

### 3. 敏感数据本地化
- API Key 存储在 `~/.uwillberich/runtime.env` (不进入工作区)
- 密码使用 macOS 钥匙串或密码管理器
- 个人信息只在本地文件中

### 4. 分离公开与私有
- 公开仓库：只包含代码、脚本、公开文档
- 本地私有：人格数据、用户信息、配置

---

## 📋 允许推送的文件

### ✅ 可以推送到 GitHub
- 代码文件 (`.py`, `.sh`, `.js`)
- 公开文档 (`.md` 除受保护外)
- 配置文件模板 (`.example`, `.template`)
- 脚本工具
- 公开的方法论文档

### ❌ 禁止推送到 GitHub
- 所有个人身份信息
- 所有 API Key 和密码
- 人格和灵魂数据
- 记忆文件
- 本地配置规则

---

## 🚨 例外情况

只有当用户**明确说**以下任一指令时，才可以推送受保护文件：
- "推送所有文件"
- "推送包括人格数据"
- "完整同步到 GitHub"
