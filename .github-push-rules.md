# Git 推送规则

_2026-03-22 创建_

---

## 📋 推送策略

### 默认行为
**不自动推送** - 所有更改只在本地提交

### 触发推送的条件
只有当用户**明确说"推送"**时，才执行 `git push`

---

## 🎯 工作流程

### 日常操作
```bash
# ✅ 做：本地提交
git add <files>
git commit -m "message"

# ❌ 不做：自动推送
# 等待用户明确指令
```

### 用户说"推送"时
```bash
# 1. 先拉取最新状态
git pull origin main

# 2. 再推送
git push origin main
```

---

## 📝 备注

- 本地提交：总是执行
- 远程推送：仅在用户明确要求时执行
- 推送前：总是先拉取最新状态
